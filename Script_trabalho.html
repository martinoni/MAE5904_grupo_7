  mutate(
    resultadoTeste = factor(resultadoTeste, levels = c("Positivo", "Negativo")),
    profissionalSaude = factor(profissionalSaude, levels = c("No", "Sim")),
    Assintomatico = as.factor(ifelse(grepl("Assintomtico",sintomas),1,0)),
    Coriza = as.factor(ifelse(grepl("Coriza",sintomas),1,0)),
    Dispneia = as.factor(ifelse(grepl("Dispneia",sintomas),1,0)),
    DistGustativos = as.factor(ifelse(grepl("Distrbios Gustativos",sintomas),1,0)),
    DistOlfativos = as.factor(ifelse(grepl("Distrbios Olfativos",sintomas),1,0)),
    DorDeCabeca = as.factor(ifelse(grepl("Dor de Cabea",sintomas),1,0)),
    DorDeGarganta = as.factor(ifelse(grepl("Dor de Garganta",sintomas),1,0)),
    Febre = as.factor(ifelse(grepl("Febre",sintomas),1,0)),
    Tosse = as.factor(ifelse(grepl("Tosse",sintomas),1,0)),
    Outros = as.factor(ifelse(grepl("Outros",sintomas),1,0)),
    verificacao = 
      as.numeric(Coriza) + as.numeric(Dispneia) + as.numeric(DistGustativos) + 
      as.numeric(DistOlfativos) + as.numeric(DorDeCabeca) + as.numeric(DorDeGarganta) + as.numeric(Febre) + as.numeric(Tosse) + as.numeric(Outros),
    verificacao2 = case_when(
      Assintomatico == 1 & verificacao >= 1 ~ 1,
      TRUE ~ 0
    ),
    tipoTeste =  case_when(
      grepl("RPIDO",tipoTeste) == TRUE ~ "Teste rpido",
      grepl("RT-PCR",tipoTeste) == TRUE ~ "RT-PCR",
      TRUE ~ "Outro"
    ),
    idade = as.integer(as.character(idade)),
    faixaetaria = cut(idade,
                      breaks = c(seq(0,60,10),Inf), right = F,include.lowest = T,
                      levels = c("0 a 9", "10 a 19", "20 a 29", "30 a 39", "40 a 49", "50 a 59", ">= 60"))
  ) %>% 
  filter(verificacao2 !=1) %>% 
  filter(tipoTeste != "Outro") %>% 
  filter(idade <= 100) %>% 
  dplyr::select(-c(verificacao,verificacao2))

Introduo

Objetivos

  • Identificar a capacidade de prevermos o resultado positivo de uma testagem para Covid-19

  • Implementar e comparar diferentes modelos de classificao vistos na disciplina

Metodologias

Nesta seo apresentado o conjunto de metodologias empregadas para classificar, segundo os sintomas, se a pessoa positivo ou negativo para covid-19.

Foram utilizados os seguintes mtodos: * Regresso Logistica * Random Forest * Suporte Vector Machine * Redes Neurais Artificiais

Mtricas de qualidade dos modelos

Foram utilizadas as seguintes mtricas

  • Sensibilidade: Probabilidade do resultado ser positivo quando de fato o individuo tem a doena.

  • Especificidade: Probabilidade do resultado ser negativo quando de fato o individuo no tem a doena.

  • Acurcia: probabilidade do teste fornecer resultados corretos, ou seja, ser positivo nos doentes e negativo nos no doentes. Expresso de outra forma a probabilidade dos verdadeiros positivos e verdadeiros negativos como uma proporo de todos osresultados

Para esse estudo, ao invs de considerarmos o individuo ter ou no ter a doena, teremos o teste RT-PCR positivo ou negativo, ficando a reinterpretao da sensibilidade e especificidade dessas mtricas conforme abaixo:

  • Sensibilidade: Probabilidade do modelo resultar em positivo quando o teste RT-PCR foi positivo.

  • Especificidade: Probabilidade do modelo resultar em negativo quando o teste RT-PCR foi negativo.

Regresso Logstica

Aqui, assumimos que o resultado do teste dadas as variveis explicativas, \(Y|x_i\), assume uma distribuio \(Bernoulli\) com probabilidade \(\pi_i\) de ser positivo. \[Y|x_i \sim Bernoulli(\pi_i)\] Alm disso, a funo de ligao : \[\log\left(\frac{\pi_i}{1 - \pi_i}\right) = \beta_0 + \beta_1x_1 + \beta_2x_2 + ... + + \beta_{13}x_{13}\]

Primeiro vamos considerar o modelo completo contendo como variveis explicativas:

  • Profissional da sade
  • Faixa Etria
  • Idade
  • Sexo
  • Sintomas (uma varivel indicadora para cada)

Agora vamos escolher o threshold. Para isso primeiro vamos definir o custo de um erro de forma heterognea: um Falso Negativo ser maior que o peso de um Falso Positivo. Isso se d pois pior dizer para um paciente infectado que ele no est com COVID (pois isso far com que ele no obedea o isolamento social e a quarentena de forma to incisiva) do que dizer para uma pessoa no-infectada que ela est com COVID (pois isso somente far com que ela realize exames adicionais que mostraro que a mesma no est infectada). No alteraremos a prevalncia dado que no temos o conhecimento tcnico epidemiolgico para tal deciso. Para a escolha do threshold usaremos o critrio de Youden com uma modificao proposta por Perkins e Schisterman, dado por: \[max(sensitivities + r \times specificities)\] Onde: \[r = \frac{1 - prevalence}{cost * prevalence}\] Sendo cost o custo relativo de um Falso Negativo comparado com o de um Falso Positivo e prevalence nmero de Positivos dividido pelo total.

Random forest

Tambm conhecida em portugus por florestas aleatrias, este algoritmo estabelece regras para tomada de deciso por meio de uma estrutura similar a um fluxograma, onde condies so verificadas, e se atendida, o fluxo segue por um ramo, caso contrrio, por outro, sempre levando ao prximo n, at a finalizao da rvore. Alm disso, este mtodo requer que seja escolhida variveis para compor cada n e isso feito aleatrio entre as variveis disponiveis para ento realizar os clculos com base nas amostras selecionadas, definindo assim qual dessas variveis ser utilizada no primeiro n. Para escolha da varivel do prximo n, novamente sero escolhidas outras variveis, desconsiderando as j selecionadas anteriormente, e o processo de escolha se repetir. Assim, a rvore ser construda at o ltimo n.

#h2o.removeAll()

ini <- Sys.time()
rforest.model.h2o <- h2o.randomForest(y=1, x=c(2,4,7:13),
                                      training_frame = train.h2o,
                                      validation_frame = valid.h2o,

end <- Sys.time()

print(paste0("Time spent was: ",round(end-ini,2)))

Redes Neurais Artificiais

O modelo de redes neurais artificiais foi implementado em python com a biblioteca Keras (Tensorflow), com o acompanhamento do treino feito pelo Comet.ml. Em sua arquitetura, tratando-se de uma Rede Neural totalmente conectada, possui uma hidden layer (com 8 unidades), 137 parmetros e funo de ativao sigmoid:

\[ \sigma(x) = \frac{1}{1 + e^x}\]

Outras caracteristicas da rede e de configurao de otimizao so:

  • Funo de custo
    • Binnary Cross-Entropy
    • \(H_P(q) = - \frac{1}{N} \displaystyle\sum_{i=1}^{N} y_i \cdot log(p(y_i)) + (1 - y_i) \cdot log(1 - p(y_i))\)
  • Otimizador
    • Adam (Padro)
  • Batch size
    • 1024 indivduos
  • Epochs
    • 100
  • Mtodo de escolha do ponto de corte
    • Youdens J statistic

Com essas caractersticas os resultados obtidos foram:

  • Treino
    • AUC: 0.67
    • Acurcia: 0.52
    • Sensibilidade: 0.81
    • Especificidade: 0.40
  • Treino
    • Acurcia: 0.55
    • Sensibilidade: 0.83
    • Especificidade: 0.36

Como esses resultados foram semelhantes aos do modelo de reresso logstica, que possui uma melhor interpretabilidade e menor complexidade de implementao (e tempo de ajuste), acabamos por no escolher as redes neurais artificiais como modelo final do projeto.
